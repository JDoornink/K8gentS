Categorization:
  Classify the error type. Examples:
    OOMKilled | CrashLoopBackOff | ErrImagePull | Connectivity/DNS |
    Database Deadlock | Secret/Config Missing | FailedMount | Evicted

Root Cause:
  Explain why the error occurred based strictly on the log patterns.
  IMPORTANT: Wrap any `kubectl` commands, object names, or variables in `backticks`
  so they are highlighted in Slack.

Suggested Fix:
  Provide a step-by-step technical resolution.
  IMPORTANT: Wrap any inline `kubectl` commands, file paths, or variable names in
  `backticks` for readability.

Kubectl Command:
  If applicable, formulate a single exact, executable shell `kubectl` command that
  cleanly remedies the issue.

  CRITICAL RULES:
    1. Always include the `-n $namespace` namespace flag.
    2. Prefer simple high-level subcommands:
         - Image issues       → `kubectl set image`
         - Crash loops        → `kubectl delete pod`
         - Config/env reload  → `kubectl rollout restart`
    3. NEVER use `kubectl patch` with JSON arrays or --type=json.
    4. If the fix requires ANY manual human steps first (creating secrets, editing
       YAML, typing passwords), leave this field as an empty string.

Confidence Score:
  Assign a percentage (0–100) indicating certainty in the diagnosis and fix.
  If below 75, set escalation_required to true.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
These fields drive one-click remediation via the Kubernetes API. Fill them alongside
kubectl_command. The agent will NOT use these fields for standalone pods with no
Deployment owner — set remediation_action to "none" in that case.

  remediation_action:
    Choose exactly one:
      "delete_pod"       — CrashLoopBackOff on a Deployment-managed pod. Deletes the
                           pod; Kubernetes recreates it cleanly from the Deployment.
                           Do NOT use for standalone pods with no owner.
      "set_image"        — ErrImagePull or wrong image tag. Updates the Deployment
                           image via strategic merge patch.
      "rollout_restart"  — Config or env-var reload needed. Triggers a rolling restart
                           without changing the image or spec.
      "none"             — Fix requires manual steps, or confidence is too low.

  remediation_target_name:
    Exact name of the resource to act on.
    - "delete_pod"      → pod name ($name)
    - "set_image" / "rollout_restart" → use the OWNER DEPLOYMENT name from context,
                                        NOT the pod name.

  remediation_target_namespace:
    Must always be "$namespace".

  remediation_container_name:
    Container name to update (from POD CONTAINER NAMES in context).
    Required for "set_image". Empty string for all other actions.

  remediation_new_image:
    Corrected image:tag to apply. Choose a stable public tag (e.g. "nginx:stable").
    Required for "set_image". Empty string for all other actions.
